# Executive Summary

The single most consequential finding is not a ranking lever. The homepage ships eleven fully-attributed client testimonials, complete with named individuals, job titles, multi-paragraph quotes, outcome metrics, engagement durations, and "months post-handover" timestamps. North Lantern Group is brand-new and sole-staffed. None of these testimonials can be defended in a sales call. They sit in `src/components/sections/Testimonials.tsx:26-184`, render to SSR HTML, and are accompanied by a status bar that asserts "11 CLIENT TESTIMONIALS // 4 SERVICE AREAS // POST-HANDOVER RESULTS". They violate the locked claim ban, expose NLG to false-advertising and PIPEDA risk, and would collapse the firm's E-E-A-T trust pillar the moment a buyer or journalist verified a single name. Remove the section before any other change ships.

After that, the homepage's biggest losses are ordinary. The title tag (42 characters) carries no service or geographic intent. The meta description (206 characters) truncates past Backlinko's measured ~155-character display ceiling, cutting the closing hook. The H1 names none of the three lanes and not the geography. Body copy mentions "Ontario" and "Ottawa" twice each, both inside footer legal copy and JSON-LD address. An Ontario buyer searching "atlassian consultant ontario" sees nothing in the ranking signals that connects this homepage to that query. The single-page architecture caps the ranking surface to one URL.

The Next.js plumbing is in good shape. SSR is solid, Vercel cache hits, security headers are enforced, sitemap and robots are honest, JSON-LD parses, schema type is correct, OG and Twitter parity are clean. Build runs green. The work is on the content, claims, and lane-targeting, not the framework.

## Technical Indexability

**Canonical drift between source declaration and SSR output.**
Severity: Medium. Type: Best practice. Confidence: High.
Evidence: `src/app/layout.tsx:31` declares `alternates.canonical = "https://www.northlanterngroup.com/"` (trailing slash). Live HTML emits `<link rel="canonical" href="https://www.northlanterngroup.com"/>` (no slash). `og:url` follows the same drift. `sitemap.xml:4` uses the trailing-slash form.
Why it matters: Google treats `/foo` and `/foo/` as distinct URLs. Three artifacts disagreeing on the canonical shape fragments PageRank consolidation and Search Console reporting.
Fix: pick the no-slash form (Next.js Metadata API normalizes to it) and update `sitemap.ts:7` and `robots.ts:13` to match. Or override the canonical via a hand-written `<link rel="canonical">` in `<head>` to keep the trailing slash everywhere. Pick one shape, align everywhere.

**Sitemap `lastmod` regenerates on every request.**
Severity: Low. Type: Best practice. Confidence: High.
Evidence: `src/app/sitemap.ts:4` uses `new Date()`; sitemap.xml line 5/11/17 confirms identical fresh timestamps.
Why it matters: Google has stated it discounts `<lastmod>` when the value updates without content changes (Google Search Central, "How Google interprets the noindex tag and lastmod").
Fix: replace with a stable per-URL constant set at build time, e.g. `new Date(process.env.VERCEL_GIT_COMMIT_DATE ?? "2026-04-24")`, or hardcode each page's last content-edit timestamp.

**Apex-to-www returns 307, not 308.**
Severity: Low. Type: Best practice. Confidence: High.
Evidence: `curl -I http://northlanterngroup.com` returns `HTTP/2 307` with `location: https://www.northlanterngroup.com/`.
Why it matters: 307 is a temporary redirect. Google has stated 301/302 differences are "very small" but for canonical-signal consolidation 308 is the correct semantic at the apex layer.
Fix: add `vercel.json` with a permanent redirect:
```json
{
  "redirects": [
    {
      "source": "/(.*)",
      "has": [{"type":"host","value":"northlanterngroup.com"}],
      "destination": "https://www.northlanterngroup.com/$1",
      "permanent": true
    }
  ]
}
```

**`robots.txt` emits a non-standard `Host:` directive.**
Severity: Low. Type: Hygiene. Confidence: High.
Evidence: `src/app/robots.ts:13` emits `host: "https://www.northlanterngroup.com"`; live `/robots.txt` line 5 shows it.
Why it matters: Only Yandex ever supported `Host:`. Google ignores it. Some validators flag it as unrecognized.
Fix: drop `host` from the return object in `robots.ts`.

**Conditional X-Robots-Tag is a deploy-time footgun.**
Severity: Low. Type: Hygiene. Confidence: High.
Evidence: `next.config.ts:34, 64-71` adds `X-Robots-Tag: noindex, nofollow` whenever `VERCEL_ENV !== "production"`. Live prod headers correctly omit the tag.
Why it matters: A deploy where `VERCEL_ENV` is unset would silently de-index the entire site.
Fix: add a post-deploy CI smoke check that fails if `X-Robots-Tag` is present on `https://www.northlanterngroup.com/`.

**`<html lang="en">` should be `lang="en-CA"` for a monolingual Canadian site.**
Severity: Low. Type: Best practice. Confidence: High.
Evidence: `src/app/layout.tsx:102` sets `<html lang="en">`. `og:locale` is `en_CA`. JSON-LD `addressCountry` is `CA`.
Why it matters: Aligns document-language signal with locale and assistive-tech expectations. SEO impact is marginal.
Fix: `<html lang="en-CA">`.

**No apple-touch-icon, no manifest, no theme-color.**
Severity: Low. Type: Hygiene. Confidence: Medium (apple-touch-icon PNG existence at `/public/brand/favicons/` not verified).
Evidence: `src/app/layout.tsx:56-62` declares only SVG and 16/16 + 32/32 PNG icons. Live HTML carries only those three icon links.
Why it matters: iOS Safari home-screen pin falls back to a screenshot. PWA install prompts unavailable. Android Chrome address-bar tinting unavailable.
Fix: add `apple: "/brand/favicons/apple-touch-icon.png"` (180x180) to the icons block. Ship `manifest.webmanifest` with name, short_name, theme_color, start_url. Add `themeColor` to metadata.

## Performance and Render Pipeline

**First Load JS for `/` is 267 kB; the contact form pulls heavy client deps into the homepage hydration bundle.**
Severity: Medium. Type: Best practice (proxy for LCP/INP). Confidence: High.
Evidence: `next build` log shows `/` First Load JS = 267 kB. `src/app/page.tsx:10` statically imports `Contact`. `Contact.tsx:1` declares `"use client"`. `Contact.tsx:4-10` imports `react-phone-number-input`, `libphonenumber-js/mobile`, `react-google-recaptcha-v3`. Why it matters: web.dev's mobile guidance puts the comfortable First Load JS budget at ~170 kB compressed; this is uncompressed and over the line. Compounds LCP and INP risk on slower mobile networks.

**Possible dead dependencies: `cobe`, `framer-motion`.**
Severity: Low. Type: Hygiene. Confidence: Medium (one subagent claimed cobe is unused, the other claimed it powers Beacon and OperationsConsole).

Fix: run the grep, decide. If unused, `npm uninstall cobe framer-motion` and verify build.

**No preconnect to reCAPTCHA origins.**
Severity: Low. Type: Best practice. Confidence: High.
Evidence: SSR HTML carries no `<link rel="preconnect">`. When `NEXT_PUBLIC_RECAPTCHA_SITE_KEY` is set, `Contact.tsx:484-489` injects the reCAPTCHA v3 script with `async, defer`.
Why it matters: 100 to 200 ms shave-off on reCAPTCHA handshake on first form interaction.
Fix: add to `layout.tsx` metadata:
```ts
other: {
  // (preconnects belong as <link> tags; if the Metadata API does not support preconnect cleanly, render them in the layout's <head> via a wrapper)
}
```
Cleaner approach: render `<link rel="preconnect" href="https://www.google.com" crossOrigin="anonymous"/>` and `<link rel="preconnect" href="https://www.gstatic.com" crossOrigin="anonymous"/>` inside the layout's `<head>` markup directly.

Severity: Low. Type: Hygiene. Confidence: High.

# Decision Register

- **Single-page architecture vs three lane URLs.** This audit recommends splitting into `/practices/atlassian-platform`, `/practices/bi-and-analytics`, `/practices/automation-and-integration`. The opposing view: a single concentrated homepage attracts fewer but stronger backlinks. Founder decision. Defer until Action 5 ships and one or two months of Search Console data inform the call.
- **Plural-team copy reframe.** This audit recommends substituting "senior-led" or "your engagement lead" for "team" in TrustStrip and Footer tagline. The opposing view: "team" reads more credibly to enterprise buyers than "solo operator". Founder decision; the proof-discipline rule pushes toward reframe, but the brand-voice cost of singular-noun framing is real.
- **Including `priceRange` in JSON-LD.** This audit recommends omitting it. Locked context bars published pricing floor. A `"$$"` placeholder leaks a positioning signal NLG has not authorized. Founder decision; recommendation is to leave blank.
- **Founder-bylined `/about` versus anonymous-firm framing.** This audit recommends Hamza-as-named-founder framing. The opposing view: a sole-founder firm with a named founder is more vulnerable to founder-key-person risk in enterprise procurement reviews. Founder decision.
- **OG image runtime: edge versus Node default.** This audit recommends Node default for static generation. The opposing view: edge runtime supports per-request OG images if a future feature ships dynamic OG content (e.g., per-essay OG cards on `/notes/<slug>`). If `/notes/` is on the roadmap, keep `runtime = "edge"` and accept the cold-start tradeoff for the homepage OG.
- **Title-tag exact wording.** This audit recommends `"Atlassian, BI and Automation Consulting in Ontario | NLG"` (56 chars). Variants worth A/B testing in Search Console after one month: `"Atlassian, Jira and BI Consulting | NLG (Ontario)"` (49 chars), `"Atlassian, BI, Automation Consulting | NLG | Ontario"` (52 chars).
